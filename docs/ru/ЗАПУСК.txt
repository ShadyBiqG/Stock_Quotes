🚀 БЫСТРЫЙ СТАРТ:

1. Двойной клик на:  setup.bat
   (Установит все зависимости автоматически)

2. Получите API ключ на https://openrouter.ai/
   (Откройте config.yaml и замените "your-openrouter-api-key-here")

3. Двойной клик на:  start.bat  или  start_web.bat


═══════════════════════════════════════════════════════════════


📁 BAT-ФАЙЛЫ ДЛЯ ЗАПУСКА:


🔧 setup.bat
   Первоначальная установка
   • Проверяет Python
   • Создает виртуальное окружение
   • Устанавливает все зависимости
   • Проверяет конфигурацию
   
   ⏱️ Время: 2-3 минуты
   🔄 Запускать: один раз при первой настройке


───────────────────────────────────────────────────────────────

📊 start.bat
   CLI версия (командная строка)
   • Анализирует котировки
   • Создает Excel отчет
   • Сохраняет в базу данных
   
   ⏱️ Время: 
      10 акций = ~45 секунд
      50 акций = ~3 минуты
      100 акций = ~6 минут
   
   📄 Результат: output\exports\2026-XX-XX_analysis.xlsx


───────────────────────────────────────────────────────────────

🌐 start_web.bat
   Веб-интерфейс (самый удобный!)
   • Открывает браузер автоматически
   • 5 интерактивных дашбордов
   • Работает на телефоне
   
   📍 Адрес: http://localhost:8501
   📱 С телефона: http://192.168.X.X:8501
   
   Страницы:
   🏠 Обзор - метрики и графики
   📈 Анализ - детальные прогнозы
   📜 История - тренды во времени
   🎯 Точность - статистика моделей
   ⚙️ Настройки - управление


───────────────────────────────────────────────────────────────

⏰ start_scheduler.bat
   Автоматический запуск по расписанию
   • Настройте расписание в config.yaml
   • Запускается в фоне
   • Логи в logs\scheduler.log
   
   Пример расписания:
   scheduler:
     enabled: true
     schedule:
       - cron: "0 19 * * 1-5"  # Будни в 19:00


───────────────────────────────────────────────────────────────

🔍 check_dependencies.bat
   Диагностика системы
   • Проверяет Python и pip
   • Проверяет все пакеты
   • Проверяет файлы проекта
   • Проверяет конфигурацию
   
   Используйте для устранения проблем


═══════════════════════════════════════════════════════════════


⚙️ ЧТО НУЖНО НАСТРОИТЬ:


1. OpenRouter API ключ (ОБЯЗАТЕЛЬНО!)
   
   📝 Откройте: config.yaml
   
   🔑 Замените:
      api_key: "your-openrouter-api-key-here"
   
   На:
      api_key: "sk-or-v1-ваш-настоящий-ключ"
   
   Получить ключ: https://openrouter.ai/
   Пополнить баланс: $10-20 для начала


2. Файл с котировками (ОБЯЗАТЕЛЬНО!)
   
   📊 Файл: Stock quotes.xlsx
   📍 Расположение: корень проекта (рядом с bat-файлами)
   
   Обязательные колонки:
   | Ticker | Price | Change | Volume |


3. Модели (опционально)
   
   📝 В config.yaml можно:
   • Выбрать другие модели
   • Убрать/добавить модели
   • Изменить temperature
   
   По умолчанию используются:
   • GPT-4 Turbo
   • Claude 3 Sonnet
   • Gemini Pro

❓ ЧАСТЫЕ ПРОБЛЕМЫ:


❌ "Python не найден"
   → Установите Python 3.10+ с python.org
   → При установке отметьте "Add Python to PATH"


❌ "API ключ не настроен"
   → Откройте config.yaml
   → Замените "your-openrouter-api-key-here" на ваш ключ


❌ "Stock quotes.xlsx не найден"
   → Поместите файл в корень папки
   → Проверьте название (точно "Stock quotes.xlsx")


❌ "ModuleNotFoundError"
   → Запустите setup.bat
   → Или вручную: pip install -r requirements.txt


❌ Медленная работа
   → Начните с 5-10 акций
   → Используйте 1-2 модели вместо 3
   → Проверьте интернет

🎯 ТИПИЧНЫЙ СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:


День 1:
  1. Запустить setup.bat (один раз)
  2. Получить API ключ и настроить config.yaml
  3. Запустить start.bat для первого анализа
  4. Проверить отчет в output\exports\


День 2+:
  • Запускать start_web.bat для удобного интерфейса
  • Просматривать дашборды
  • Отслеживать историю трендов


Для автоматизации:
  • Настроить расписание в config.yaml
  • Запустить start_scheduler.bat
  • Получать отчеты автоматически

📱 ДОСТУП С ТЕЛЕФОНА:


1. Запустите start_web.bat на компьютере

2. На компьютере узнайте IP:
   • Windows: ipconfig
   • Найдите IPv4: 192.168.X.X

3. На телефоне в том же Wi-Fi откройте:
   http://192.168.X.X:8501

4. Добавьте в закладки!
